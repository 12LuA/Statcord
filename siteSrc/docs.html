API Documentation

Base URL: https://disstat-api.tomatenkuchen.com/v1/

Authentification can be done using the Authorization header, or the apikey query parameter. The API key can be found on your dashboard and is the same for all of your bots.

Getting bot data

This endpoint returns the basic data DisStat has stored about the bot, like it's username and avatar.
If the bot is private, an API key authorized for the bot must be provided in the Authorization header.

Method: GET
URL: https://disstat-api.tomatenkuchen.com/v1/bot/<Bot ID>

Headers:

Name Type Description Required
Authorization string API key authorized for the bot from the DisStat dashboard. Only if the bot is private.

Query:

Name Type Description
returnStats boolean Whether the stats should be returned. Defaults to false, set to true to enable.
groupSize number Amount of data points to return.
start number The start date to filter the data by. Defaults to no start date.
end number The end date to filter the data by. Defaults to no end date.

Posting bot data

This endpoint returns the basic data DisStat has stored about the bot, like it's username and avatar.

Method: POST
URL: https://disstat-api.tomatenkuchen.com/v1/bot/<Bot ID>

Headers:

Name Type Description Required
Authorization string API key authorized for the bot from the DisStat dashboard. Yes

Body:

Name Type Description
returnStats boolean Whether the updated stats should be returned. Defaults to false.
servers number The amount of servers (guilds) the bot is in
users number The amount of users the bot can see across all servers
shards number The amount of shards the bot is running with
apiPing number The time in milliseconds how long a ping from/to Discord takes
ramUsage number The amount of memory/RAM the bot's process is using currently, in bytes
ramTotal number The amount of memory/RAM the bot could use theoretically, in bytes
cpu number The current cpu usage of the bot's process
bandwidth number The current bandwidth usage of the bot's process, in bytes

Posting a ran command

Stores a ran command for the bot. This is used to track the most used commands. You should remove the prefix and arguments and only send the command name. Also, custom commands shouldn't be sent to protect user privacy.

Method: POST
URL: https://disstat-api.tomatenkuchen.com/v1/bot/<Bot ID>/command

Headers:

Name Type Description Required
Authorization string API key authorized for the bot from the DisStat dashboard. Yes

Body:

Name Type Description Required
command string The command name without a prefix. Yes
user string The user ID or another per-user identifier to track the amount of engaged users No
guild string The guild ID or another per-guild identifier to track the amount of active guilds No
